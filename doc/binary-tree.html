Binary Tree

Lehi Toskin

 (require binary-tree) package: binary-tree
This library provides functionality for creating and modifying binary trees.
There is also a function provided for graphic creation via tree->pict.
The following are provided by binary-tree and binary-tree/base.

1 Basic Structures

This library has two structs that it uses for everything

struct

(struct btree (root <? =?)
    #:mutable)
  root : node?
  <? : (-> any/c any/c boolean?)
  =? : (-> any/c any/c boolean?)

struct

(struct node (data left right)
    #:mutable)
  data : any/c
  left : (or/c node? null?)
  right : (or/c node? null?)

2 Procedure List

2.1 Constructors

procedure

(make-tree data <?) → btree?

  data : any/c
  <? : (-> any/c any/c boolean?)

procedure

(make-tree-eq data <?) → btree?

  data : any/c
  <? : (-> any/c any/c boolean?)

procedure

(make-tree-eqv data <?) → btree?

  data : any/c
  <? : (-> any/c any/c boolean?)

procedure

(make-custom-tree data <? =?) → btree?

  data : any/c
  <? : (-> any/c any/c boolean?)
  =? : (-> any/c any/c boolean?)
Procedures to make a new binary tree. make-tree, make-tree-eq, and make-tree-eqv compare elements with equal?, eq? and eqv? respectively.

<? and =? should reflect the type of data being fed into the tree (i.e. string<? or string=?).

2.2 Modifiers

procedure

(tree-add tree val ...) → btree?

  tree : btree?
  val : any/c

procedure

(tree-add! tree val ...) → btree?

  tree : btree?
  val : any/c
Takes at least one variable and adds it to tree.

procedure

(tree-cons t1 t2) → btree?

  t1 : btree?
  t2 : btree?
Adds t2 to the rightmost position of t1.

procedure

(tree-append t1 t2) → btree?

  t1 : btree?
  t2 : btree?

procedure

(tree-append! t1 t2) → btree?

  t1 : btree?
  t2 : btree?
Adds the contents of t2 to t1.

procedure

(tree-mirror tree) → btree?

  tree : btree?
Swaps (inverts) each left and right child in the tree.

procedure

(tree-remove tree val) → btree?

  tree : btree?
  val : any/c

procedure

(tree-remove! tree val) → btree?

  tree : btree?
  val : any/c
Removes val from tree. Does nothing if tree does not contain val.

2.3 Accessors

procedure

(tree-equal? tree) → boolean?

  tree : btree?

procedure

(tree-eq? tree) → boolean?

  tree : btree?

procedure

(tree-eqv? tree) → boolean?

  tree : btree?
Checks if the tree has been created using that constructor.

procedure

(tree=? t1 t2) → boolean?

  t1 : btree?
  t2 : btree?
Checks both the contents and the structure of the trees for equality.

procedure

(tree-contains? tree val) → boolean?

  tree : btree?
  val : any/c
Checks if the tree contains val.

procedure

(tree-copy tree) → btree?

  tree : btree?
Returns a copy of tree.

procedure

(in-tree tree) → sequence?

  tree : btree?
Returns a sequence to iterate over tree.

procedure

(tree->list tree) → list?

  tree : btree?
Returns a list containing the elements of tree sorted by <?.

procedure

(tree-length tree) → nonnegative-integer?

  tree : btree?
Returns the number of elements inside tree.

procedure

(tree-branch tree val) → btree?

  tree : btree?
  val : any/c
Returns the branch of the tree with the root val. If btree does not contain val, a new tree is returned.

procedure

(tree->pict tree    
  [#:color color    
  #:style style]) → pict?
  tree : btree?
  color : string? = "black"
  style : procedure? = naive-layered
Returns a pict with the data of each node (printed with ~v), modified by a color string, and ordered by the style procedure. The style procedure must be one of naive-layered, binary-tidier, or hv-alternating.

Provided by binary-tree and binary-tree/graphics.
